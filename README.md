# UniqueBlog — System Analysis & Engineering

Командный проект по системной и программной инженерии. Полный цикл разработки веб-платформы для ведения блогов — от анализа требований до тестирования и документирования.

## О проекте

UniqueBlog — веб-приложение для публикации постов с категориями, комментариями и модерацией контента. Стек: Python, Django, SQLite, Bootstrap, PythonAnywhere.

## Что сделано

### Анализ требований
- Сформулировал **user story** в формате "Как [роль] / Я хочу / Чтобы" с acceptance criteria
- Описал **функциональные и нефункциональные требования**
- Составил **матрицу требований** с критериями проверки

### Моделирование системы
- **Use Case диаграмма** — сценарии взаимодействия пользователей с системой
- **Диаграмма последовательности** — ключевые сценарии работы
- **Диаграмма классов** и **диаграмма объектов**
- **IDEF0 диаграммы** — контекстная диаграмма и декомпозиции
- **DFD диаграммы** — потоки данных (посты, комментарии)
- **Логическая модель БД** — нормализованная ER-модель

### Архитектура
- Описал и обосновал клиент-серверную архитектуру
- Построил архитектурную диаграмму
- Отразил соответствие требований компонентам архитектуры

### Техническое задание
- Разработал **ТЗ по ГОСТу** на систему

### Организация разработки
- Выбрал методологию Waterfall с обоснованием через сравнительную таблицу
- Настроил Git-репозиторий и инструменты разработки
- Создал документацию разработчика через Sphinx
- Создал документацию пользователя в Confluence

### Тестирование
- Разработал **12 тест-кейсов** (позитивные и негативные сценарии)
- Провёл тестирование всех функциональных требований
- Составил **матрицу рисков** с планами реагирования
- Провёл анализ технологических угроз

## Инструменты

`draw.io` `Confluence` `GitHub` `Sphinx` `Django` `SQLite` `Bootstrap` `PythonAnywhere`
